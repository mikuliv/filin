# Датасеты Филин

В этой папке хранится только описание структуры датасетов и служебные пустые файлы.

Большие датасеты, PCAP, сырые логи, JSONL-файлы, CSV-выгрузки, модели и артефакты обучения не хранятся в Git. Они сохраняются локально, размещаются в защищенном хранилище или публикуются отдельно с описанием версии, источника и методики сбора.

Рекомендуемая локальная структура:

```text
filin/datasets/
  raw/          # сырые логи и PCAP
  processed/    # нормализованные JSONL/CSV
  reports/      # отчеты по прогонам
  examples/     # маленькие учебные примеры CSV
```

Перед публикацией датасета нужно проверить отсутствие секретов, внешних адресов, персональных данных и лишних технических артефактов.

## Признаки для моделей

Сырые события лабораторного стенда не являются готовыми признаками для модели. Для обучения используются агрегированные датасеты:

```text
raw events -> normalized events -> feature extraction -> windows.csv / flows.csv -> training
```

`windows_v0_1.csv` содержит признаки по временным окнам наблюдения. `flows_v0_1.csv` в версии v0.1 является прототипом flow-level представления и будет расширен после подключения Zeek/Suricata.

Поля `scenario_id`, `run_sequence` и planned time используются только для разметки и анализа, но не являются входными признаками модели.

Маленькие учебные примеры находятся в `filin/datasets/examples/`. Большие CSV-выгрузки должны оставаться в `filin/lab/output/` или внешнем хранилище и не коммитятся в Git.
