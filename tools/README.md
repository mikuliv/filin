# Инструменты проекта

`tools/` содержит локальные CLI для аудита, документации, проверки артефактов
и вспомогательной обработки. Скрипты не должны обращаться к внешней сети или
изменять frozen evidence без явно разрешённого этапом процесса.

Основные точки входа:

- `tools/docs/` — генераторы и валидаторы документации;
- `tools/audit/` — проверки evidence bundles и исключения runtime artifacts;
- `tools/check_release_images.py` — проверка release-образов лаборатории.

Команды проверки перечислены в
[руководстве по тестированию](../docs/getting-started/testing.md).
