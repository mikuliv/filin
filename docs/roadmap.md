# Дорожная карта

## 1 год

- Рефакторинг исходной идеи Anomalyzer в самостоятельную архитектуру Филин.
- Описание доменной модели сетевого события, признаков, предсказания и инцидента.
- Прототип backend на FastAPI.
- Подготовка in-memory repository и контракта для будущего PostgreSQL или Elasticsearch.
- Начальная документация архитектуры и лабораторного стенда.

## 2 год

- Развитие обработчика инцидентов.
- Расширение сопоставления MITRE для разных классов атак.
- Прототип генерации Sigma-кандидатов.
- Развертывание стенда: Kali Linux, целевой узел, Zeek или Suricata, collector, Elasticsearch, backend.
- Подготовка сценариев атак и базовой методики проверки правил.

## 3 год

- Эксперименты с моделями и признаками.
- Сравнение MLP, RandomForest, XGBoost/LightGBM и AutoEncoder.
- Оценка ложных срабатываний и пропусков атак на лабораторных и открытых датасетах.
- Оформление ВКР.
- Подготовка демонстрационного сценария: событие -> детекция -> инцидент -> MITRE -> Sigma -> проверка.

## Следующий этап лабораторного стенда

Docker-режим v0.2 выполняет реальные безопасные действия между контейнерами и формирует наблюдения `traffic-client`. Следующий этап: подключение Zeek/Suricata для независимых conn/http/dns flow, сопоставление их с run metadata и проверка качества датасета.

## Филин v0.2.1 — Docker-to-Docker evaluation

Выполнены сравнения независимых Docker runs, анализ стабильности признаков и контроль переноса baseline-моделей. Следующая исследовательская задача остаётся прежней: независимое наблюдение через Zeek/Suricata и увеличение объёма разнообразных данных.
