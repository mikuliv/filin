# Технический аудит происхождения кода

## Методика

Сравнение выполнено между всеми файлами `Anomalyzer-main/` и `filin/`:

1. SHA-256 точного содержимого;
2. SHA-256 текста после удаления строковых комментариев и whitespace для `.py`, YAML, JSON, Markdown и text files;
3. структурное попарочное сравнение нормализованных Python-файлов (`SequenceMatcher`, порог 0.45);
4. `git log --follow`, `git log --diff-filter=R`, name-status, author history и `git blame` ключевых файлов;
5. поиск imports, path references, execution commands, model/scaler names, ONNX и CICIDS references.

## Пограничные места для дальнейшей проверки

- `filin/ml/configs/inference.yaml` описывает local ONNX/scaler paths. Это не ссылка на historical artifact, но перед distribution следует закрепить происхождение каждой подставленной модели.
- Backend, FastAPI, Docker Compose и ML patterns могут быть концептуально похожи на типовые решения; автоматическое сравнение не доказывает отсутствие заимствований идей или внешних не-Git источников.
- Git author у всех доступных commits один. Это не снимает риск служебного произведения, неотражённых contributors или прав третьих лиц.

## Ограничения метода

Аудит не может обнаружить удалённую до commit историю, локальные непубликовавшиеся исходники, копирование с сильной ручной переработкой или правовой характер творческого сходства. Результат применим как техническое доказательство отсутствия обнаруженных текстовых и runtime-заимствований, а не как юридическая экспертиза.
