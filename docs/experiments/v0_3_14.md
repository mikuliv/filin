# Филин v0.3.14 — shadow-readiness и passive integration audit

Технический этап воспроизвёл 700 immutable prediction records v0.3.13 без загрузки модели. Сформировано 935 semantic events: 700 observations, 124 alerts, 12 reviews и 99 агрегатов для 212 continuation rows.

Все 935 events прошли schema/privacy validation; 13/13 invalid fixtures отклонены. Девять replay-профилей дали одинаковые event-set, sequence, alert, review и hash-chain SHA-256. At-least-once replay дал 1870 delivery attempts, 935 duplicates и 935 sink-unique events без collisions или semantic duplicates.

Bounded queue имеет capacity 2048 и canonical peak 935. Stress-тесты достигли high/critical watermarks, учли drop и сохранили alert priority. Checksum spool, восемь crash points, retry, пять rate-limit profiles и 22 fault scenarios пройдены. Automatic action, network block, backend write, production connection, first-alert loss, review loss, causal violations и unaccounted drops равны нулю.

Расхождение frozen `checkpoint_resume_passed=false` классифицировано как `policy_flag_stale`: заключительный v0.3.13 resume evidence создан после frozen policy и подтверждает reuse без повторной prediction. Старые файлы не изменялись.

v0.3.14 разрешает только v0.3.15 controlled passive shadow trial в изолированной среде. Shadow mode, backend integration и production readiness остаются `false`.
