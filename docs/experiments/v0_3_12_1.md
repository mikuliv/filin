# Филин v0.3.12.1 — технический аудит задержки и покрытия

Этап выполнен как read-only post-hoc аудит завершённой regression v0.3.12. Кандидат `v0311:19176acb401be2d4`, его manifest, validation lock и обе immutable regression predictions совпали с frozen SHA-256. Fit, calibration, conformal fit, threshold selection, prediction generation, Docker, Zeek и feature extraction не выполнялись.

## Главный результат

Frozen показатели `0.733333` воспроизведены: v0.3.9 даёт 12/10/8 alert episodes в первом/втором/третьем положении массива записей, v0.3.10 — 23/21/16. Но массив immutable prediction детерминированно упорядочен не по causal order. После восстановления порядка по frozen `causal_order` фактические alert windows равны 29/1/0 для v0.3.9 и 60/0/0 для v0.3.10. Все 24 кажущихся third-window задержки — `input_or_mapping_error` в episode latency evaluator, а не задержка model, conformal, strong/weak policy, activity key или state machine.

Earliest policy eligibility и actual emission совпали в одном causal окне для всех 90 attack episodes. State-machine extra delay, потеря pending, подавление первого alert, смена activity key и неожиданные transitions равны нулю. Это диагностическое уточнение не переписывает immutable records, frozen metric, gate `0.75` или отрицательный policy result v0.3.12.

## Дискретность gate

Frozen числители равны 22/30 и 44/60. Шаг одного эпизода — `0.033333` и `0.016667`; для математического прохождения обоим benchmark нужен один дополнительный detected episode. Следующие достижимые rates — `0.766667` и `0.750000`. Статистическое сравнение является только диагностическим.

## Историческое покрытие

Для v0.3.8 цепочка равна: 252 marker/PCAP intervals → 252 extracted rows включая 36 warm-up → 216 scored/mapped/predicted/reported rows. Первый divergence — штатное исключение warm-up; frozen lock изначально фиксирует 216 строк и 72 эпизода. Причина прежнего ожидания 252 классифицирована как `expected_count_metadata_error`; v0.3.8 не переоценивался.

Для v0.3.6 и v0.3.7 подтверждено отсутствие неизменяемой frozen 51-feature table. Сохранившиеся PCAP, Zeek и иные источники означают `rebuildable_but_not_frozen`, но не делают наборы пригодными для научной regression. Содержимое PCAP не анализировалось, Zeek не запускался.

## Итог

Созданы стандарт хранения regression bundle, tracked manifest template и strict validator. Следующий допустимый научный этап — новый training/internal-validation cycle на полностью новых данных и seeds. `v0312_regression_passed=false`, readiness к v0.3.13, shadow mode и backend integration остаются `false`.
