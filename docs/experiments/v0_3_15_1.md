# v0.3.15.1 — усиление доказательной базы пассивного runtime

Этап является corrective technical audit без обучения модели. Frozen candidate, 51-feature schema, thresholds, predictions и исторические bundles не изменялись.

## Предварительный аудит

Подтверждены fallback неизвестного fault-сценария в healthy, unconditional passed, раздельные spool/checkpoint demonstrations, нестрогий ACK, retry всех exceptions, hardcoded drop zero, неполный resume, формальные performance profiles, неполный privacy target set, report-only tests и отсутствие collectors в CI.

## Исправления

Реализован один at-least-once runtime path: validation → privacy → canonical serialization → durable fsync spool → bounded priority queue → token bucket → real batch send → strict ACK → typed retry/permanent decision → atomic checkpoint → compaction → reconciliation. Добавлены explicit fault registry, crash recovery, strict bundle lock, runtime resource monitor и behavioral tests.

## Reassessment

Исторические v0.3.14/v0.3.15 артефакты сохранены. Bundle v0.3.15 доступен локально и все его manifest hashes совпали. Однако исходный v0.3.15 code path не создаёт достаточную raw behavioral evidence для точных claims о retry, ACK faults и checkpoint restore. Исправленный technical replay показывает работоспособность нового runtime, но не заменяет историческое выполнение.

## Решение

Corrective remediation завершена, но `v0315_runtime_claims_revalidated=false` и `v0315_readiness_decision_reconfirmed=false`. Поэтому `candidate_ready_for_v0_3_16_staging_connector_readiness=false`. Production, shadow mode, backend integration и automatic enforcement остаются запрещены.
