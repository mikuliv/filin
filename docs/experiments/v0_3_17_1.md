# Филин v0.3.17.1 — корректирующий аудит доказательств, временных трасс и производительности

## Статус протокола

Протокол заморожен до внесения корректировок и до первого prospective-события. Исходный HEAD: `43922e06050f53c91b9195af63c5d55325904eca`. Этап v0.3.17 сохраняет статус `completed` с отрицательным policy result и не переоценивается.

Активная рабочая копия и тяжёлый runtime размещены на профиле `samsung_860_evo_ntfs_v03171`. Локальные абсолютные пути задаются только runtime-профилем и не входят в tracked evidence. Изменение среды хранения означает, что результаты производительности v0.3.17.1 нельзя напрямую сравнивать с v0.3.17.

## Назначение

Этап должен установить и, где это допустимо, устранить подтверждённые причины отрицательного результата v0.3.17:

1. десять historical-anchor mismatches;
2. нарушения clock-domain и связности временных трасс;
3. неоднозначную семантику transport duplicate counters;
4. недостоверную агрегацию latency и composite performance policy;
5. два пропущенных случая corruption suite;
6. ошибку `LOCK_PATH is not defined` и неполную штатную финализацию.

Кандидат, feature contract, event contracts, state policy, thresholds и historical evidence остаются неизменными. Обучение, калибровка, conformal fit и подбор порогов запрещены.

## Классификация предполагаемых изменений

Начальная классификация — `T+I`:

- `T`: anchor validator, report generation, metric aggregation, corruption fixtures, bundle validator, finalizer и документация;
- `I`: versioned timing trace, clock/boot identity и attempt/ACK linkage.

На момент заморозки:

- `tooling_changed=true`;
- `instrumentation_changed=true`;
- `runtime_delivery_path_changed=false`;
- `full_endurance_rerun_required=false`.

Если реализация затронет outbox, journal, batching, retry, ACK, checkpoint, compaction, receiver commit, event generation или state semantics, изменение переклассифицируется в `R`, `full_endurance_rerun_required=true`, а 45-минутный trial будет заменён коротким technical preflight. Четырёхчасовой endurance trial автоматически не запускается.

## Prospective-план

Если `runtime_delivery_path_changed=false`, выполняются три независимых label-free run общей фактической длительностью 45 минут:

- Run A — timing nominal, 15 минут;
- Run B — retries and restart, 15 минут;
- Run C — backlog and recovery, 15 минут.

Используются новые run/session/trace/attempt/batch IDs, seeds, certificates, process и container identities. События v0.3.17 не используются как prospective input. Scientific accuracy, F1 и recall не рассчитываются.

Если `runtime_delivery_path_changed=true`, выполняется только 10–15-минутный technical preflight, readiness к v0.3.18 остаётся false и требуется отдельный endurance trial.

## Итог

Этап завершён с положительным corrective policy result. Официальным является только revision 3:

- три независимых запуска общей длительностью `2700.0254642` секунды;
- `33753` logical events и `438899` timing trace rows;
- clock attestation пройдена, linear violations, linkage errors, wrong-attempt ACK links и stale timestamp reuse равны нулю;
- source/connector/receiver sets совпадают, pending events, semantic duplicates, collisions, unaccounted drops и final backlog равны нулю;
- sensor→receiver p95 `2.8295 ms`, p99 `5.691834 ms`;
- connector ingress ACK p95 `1.55421 ms`, connector→receiver p95 `1.32791 ms`;
- durable throughput `12.500993 events/s`;
- corruption suite `20/20`, clean finalization `9/9`, bundle validator и strict resume пройдены.

Run A выполнил nominal workload без retries. Run B подтвердил 22 retry/duplicate-status ответа, receiver restart и certificate reconnect без двойного учёта. Run C подтвердил временное замедление, ограниченный backlog и полное восстановление до `final_backlog=0`.

Delivery path не изменён, поэтому новый четырёхчасовой endurance trial не требуется и не запускался. Рабочая среда хранения перенесена на SSD; показатели этого этапа нельзя напрямую сравнивать с HDD-результатами v0.3.17. Разрешён только design review v0.3.18.
