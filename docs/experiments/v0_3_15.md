# Филин v0.3.15 — локальный controlled passive shadow trial

Этап проверяет непрерывный локальный путь frozen candidate v0.3.11 от закрытия capture до подтверждения локальным mock sink. Это не production shadow mode и не разрешение автоматических действий.

## Замороженный дизайн

До первой фактической Docker-session зафиксированы protocol, десять session ID и seeds, Latin-square episode schedule, двадцать benign variants, fault/restart schedules, feature schema, blind-label, metric, stability, readiness и resource policies. Фактическая длительность capture-window `1.0` секунды извлечена из авторитетного построителя `build_network_sensor_v4_dataset.py`, совместимого с frozen candidate.

## Кампания

Выполнены 10 последовательных sessions в пяти группах: baseline endurance, burst/jitter, recovery overlap, sink fault и restart/resume. Получено 1520 уникальных canonical PCAP, 80 warm-up и 1440 scored windows. Сценарий содержит 40 benign и 40 attack episodes; каждый attack class представлен восемью episodes, каждый benign variant — двумя.

Каждый PCAP обрабатывался контейнеризированным Zeek после закрытия. Из трафика причинно извлекались 51 frozen feature, затем выполнялся один online inference. Labels были доступны только после фиксации predictions, semantic events, полного drain и pre-label lock.

## Устойчивость

Проверены temporary unavailable, timeout, rate limit, connection reset, slow consumer, duplicate/malformed ACK, exporter/sink/feature worker/sensor restart и две crash boundaries. Повторный transport send не создавал semantic duplicate. Backend writes, production/external connections, automatic actions и network blocks отсутствовали.

## Результат

Все scientific, stateful, episode, class/session/group/variant/length, calibration, conformal, availability, latency, lag, recovery, privacy и fail-safe gates пройдены. Разрешён только следующий технический этап v0.3.16 staging connector readiness. Shadow mode, backend integration и production readiness остаются запрещёнными.
