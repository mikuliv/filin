# Филин v0.3.12.2 — causal-order corrected frozen regression

Этап повторно оценивает неизменный кандидат v0.3.11 на научном знаменателе из трёх frozen bundles: v0.3.8, v0.3.9 и v0.3.10. Обучение, калибровка, conformal refit, подбор порогов и feature selection запрещены. Исторический результат v0.3.12 остаётся неизменным и отрицательным.

Для v0.3.8 восстановлена доказанная размерность: 6 runs, 252 извлечённых интервала, 36 warm-up и 216 scored rows, 72 эпизода, 51 признак. Выполнен ровно один новый predict. Прогнозы v0.3.9 и v0.3.10 повторно не вычислялись и подключены по frozen SHA-256.

Все stateful и episode-метрики вычислены только после сортировки `(benchmark_id, run_id, activity_key, causal_order, immutable_row_id)`. Инвариантность подтверждена для reverse, трёх seeded row shuffles и group-block shuffle. Исторический физический порядок сохранён только как неполитический контроль: v0.3.9 — `12/10/8`, v0.3.10 — `23/21/16`. Исправленный causal order дал соответственно `29/1/0` и `60/0/0` с detection by second window `1.0`.

Оконные результаты: v0.3.8 и v0.3.9 имеют macro F1 `0.9907335907`, balanced accuracy `0.9907407407`, benign recall `1.0`, FPR `0.0`, attack macro recall `0.9888888889`; v0.3.10 имеет единичные recall/F1 и нулевой FPR. На всех трёх bundles attack episode recall и episode alert precision равны `1.0`, benign episode FAR и unresolved pending rate равны `0.0`.

Scientific coverage, absolute, stateful, episode, aggregate, calibration, conformal и non-inferiority gates пройдены. Поэтому `v03122_regression_passed=true` и кандидат разрешён только для prospective blind holdout v0.3.13. Shadow mode, backend integration и production остаются запрещены.

Runtime-отчёты находятся в `ml/reports/v0_3_12_2/` и исключены из Git. В Git сохраняются протоколы, политики, код, тесты и три строгих regression bundle manifest.
