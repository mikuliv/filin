# v0.3.15.5.1 — восстановление runtime-контракта кандидата

## Заморозка протокола

Этап является узким prospective runtime-only trial для неизменного кандидата `v03154:65a3dd912d845bc1`. Протокол revision 1 заморожен до реализации `shadow_event_v2` и до первого capture. Научные метрики не пересчитываются, метки не создаются и не используются, обучение, калибровка, conformal fit, подбор признаков и порогов запрещены.

Предложенные исходным заданием seeds `22101–22403` пересекались с официальной кампанией v0.3.15.5. До первого capture они заменены на свободные `23101–23403`. Идентификаторы двенадцати сессий сохранены без изменений.

## Научная опора

Единственная научная опора — неизменный valid holdout v0.3.15.5. Его общий результат остаётся отрицательным: `v03155_independent_holdout_passed=false`, кандидат не был promoted из-за несовместимости frozen `shadow_event_v1`. Отдельные scientific subpolicies кандидата прошли и могут участвовать только в композиции с новой независимой runtime evidence.

## Контракты

`shadow_event_v1` остаётся историческим неизменяемым контрактом с SHA-256 `cc6df6cf93e5dabeafe147ebeb232d04f7a82805e8ddb8409987f44b5cca08fe`. Автоматическая миграция v1 запрещена. Новый `shadow_event_v2` разделяет структурную JSON Schema validation и авторизацию кандидата через отдельный frozen registry с canonical commitment.

## Кампания

Заморожены четыре группы по три последовательные независимые сессии: contract baseline, transport faults, crash/resume и load/clock. Каждая сессия содержит 200 новых capture windows, из них 10 warmup и 190 scored; всего 2 400 captures и 2 280 label-free immutable predictions. Используются два реальных exporter worker, batch 50, bounded queue, token bucket, durable spool, strict ACK, atomic checkpoint и локальный mock sink.

## Критерии и ограничения

Положительный результат требует прохождения contract, registry, identity, independence, no-fit, integrated runtime, reconciliation, 12/12 faults, crash recovery, hash chain, ACK, exact latency, performance, resources, privacy, strict resume, corruption и bundle gates. Даже положительный результат разрешает только переход к разработке изолированного staging-only этапа v0.3.16. Shadow mode, backend integration, production, внешние соединения, автоматические действия и изменение backend остаются запрещены.
