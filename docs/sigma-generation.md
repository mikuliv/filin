# Генерация Sigma

Sigma в Филин используется для подготовки черновиков правил обнаружения по карточке инцидента.

Sigma хорошо подходит для логов и SIEM-событий: Windows Event Log, Sysmon, Zeek, Suricata, proxy, DNS, web logs и нормализованные события безопасности.

Не любое flow-событие можно напрямую выразить через Sigma. Если в событии нет нормализованных полей, контекста источника, понятного `logsource` и стабильных признаков, правило будет слишком слабым или непереносимым.

## Принцип работы прототипа

1. Обработчик инцидентов формирует карточку.
2. Генератор Sigma берет класс, уровень риска и доступные сетевые поля.
3. Генерируется правило со статусом `experimental`.
4. Валидатор правил имитирует первичную проверку.
5. Аналитик проверяет правило на лабораторном стенде.

## Важное ограничение

Sigma-правило является кандидатом и требует проверки на тестовом стенде перед применением в SIEM или демонстрационном сценарии.
