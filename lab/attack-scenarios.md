# Сценарии атак

Первые сценарии для стенда:

- контролируемое сканирование портов только по allowlist внутренних сервисов;
- контролируемые ошибки авторизации на тестовой учетной записи;
- безопасный web-probe без реальных вредоносных payload-ов;
- низкоинтенсивная имитация всплеска трафика внутри лабораторной сети;
- beacon-симуляция через периодические обращения к внутреннему control-api.

Все сценарии должны иметь лимиты длительности и интенсивности. Внешние цели запрещены.
